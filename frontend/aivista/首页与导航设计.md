# AiVista 首页与导航设计

> 对应前端迭代：前端迭代 1——应用壳层与用户认证  
> 状态：前端迭代 1 已完成；生成、资产与灵感真实数据待后续迭代接入。

## 1. 目标与范围

为 AiVista Web 建立统一的应用壳层，使用户始终可以在灵感、生成和资产之间切换，并从侧栏完成登录或进入个人主页。

本次包含：

- 左侧固定导航栏及移动端底部导航；
- 首页的创作入口区和灵感内容区布局；
- 登录页、会话恢复、头像入口和个人主页；
- 生成页与资产页的独立入口和空状态。

本次不包含：真实文生图、局部重绘、真实资产列表、真实灵感流、收藏、发布、点赞和关注。

## 2. 路由

| 路由 | 页面 | 渲染策略 | 当前状态 |
| --- | --- | --- | --- |
| `/` | 灵感首页 | 公开页面；当前静态布局，后续接入 SSR/数据缓存 | 已实现布局 |
| `/generate` | 生成工作台 | 客户端交互为主 | 已实现入口与空状态 |
| `/assets` | 个人资产 | 客户端请求为主，需登录 | 已实现入口与登录提示 |
| `/profile` | 个人主页 | 客户端请求为主，需登录 | 已实现资料展示、昵称/简介编辑与退出登录 |

## 3. 页面布局

```text
桌面端
┌──────────┬──────────────────────────────────────────────────┐
│ 品牌标识 │ 创作入口区：标题、生成入口、能力说明              │
│          ├──────────────────────────────────────────────────┤
│ 灵感     │ 灵感内容区：筛选栏、搜索、作品卡片网格            │
│ 生成     │                                                  │
│ 资产     │                                                  │
│          │                                                  │
│ 登录/头像│                                                  │
└──────────┴──────────────────────────────────────────────────┘

移动端：侧栏收起为底部导航；内容区域使用单列或双列网格。
```

首页的两个主内容区对应设计参考中的上、下红框；左侧红框对应全局导航栏。视觉采用低饱和云白背景、深石墨文字与青蓝色创作强调色，作品卡片使用渐变占位以避免在未接入真实图片前伪造业务数据。

## 4. 交互与认证

| 场景 | 行为 |
| --- | --- |
| 点击灵感/生成/资产 | 灵感可直接进入 `/`；生成、资产在已登录时进入对应路由，未登录时停留当前页并打开登录卡片。 |
| 未登录点击登录 | 在当前页面上方打开登录/注册悬浮卡片，不跳转路由。 |
| 登录卡片位置 | 遮罩覆盖当前视图，卡片始终在视图水平、垂直中心；窄屏保留安全边距。 |
| 提交登录 | 调用 `POST /api/auth/login`；成功后保存 Access Token 于内存、后端写入 Refresh Token Cookie，关闭卡片并更新侧栏头像。 |
| 注册账号 | 在同一张卡片切换为注册表单；需确认两次密码一致后调用 `POST /api/auth/register`，成功后回到登录表单。 |
| 密码可见性 | 登录密码、注册密码和确认密码输入框均提供右侧眼睛按钮，可独立切换明文或隐藏状态。 |
| 页面刷新或重新打开 | 调用 `POST /api/auth/refresh`，成功后携带新 Access Token 调用 `GET /api/users/me` 恢复用户资料。 |
| 已登录状态 | 侧栏底部显示头像；无头像时显示昵称首字；点击进入 `/profile`。 |
| 退出登录 | 调用 `POST /api/auth/logout`，清空内存登录态，返回首页。 |
| 编辑个人资料 | 在 `/profile` 编辑昵称和简介，调用 `PUT /api/users/me`；头像修改等待媒资上传能力。 |
| 资产页未登录 | 若通过直接 URL 访问，展示登录引导且不请求资产数据；点击引导按钮打开登录卡片。 |

登录失败时展示后端返回的统一错误消息；业务请求收到 `40100` 时由全局 API Client 单飞刷新并重放，收到 `40102` 时清理会话并打开当前页面的登录卡片。

## 5. API 映射

| 前端行为 | 后端接口 |
| --- | --- |
| 登录 | `POST /api/auth/login` |
| 注册 | `POST /api/auth/register` |
| 会话恢复 | `POST /api/auth/refresh` |
| 获取当前用户 | `GET /api/users/me` |
| 修改当前用户资料 | `PUT /api/users/me` |
| 退出登录 | `POST /api/auth/logout` |

所有浏览器请求使用同域 `/api` 相对路径；开发环境由 Next.js rewrite 转发到 Spring Boot，生产环境由 Nginx 代理。

## 6. 验收标准

- 桌面端显示左侧导航、创作入口区与灵感内容区；移动端显示底部导航。
- 点击三项导航可进入对应路由，并正确显示当前激活状态。
- 未登录时侧栏显示登录按钮，点击在当前视图中央打开登录卡片。
- 使用有效后端账号登录后，侧栏显示头像或昵称首字；刷新页面后仍可通过 Refresh Cookie 恢复登录态。
- 点击头像进入个人主页；退出登录后恢复登录按钮。
- 个人主页可保存昵称与简介，保存后侧栏和资料页同步显示新昵称。
- `pnpm lint` 和 `pnpm build` 通过。
