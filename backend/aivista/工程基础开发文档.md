# AiVista 工程基础开发文档

> 对应迭代：迭代 0——工程基础（已完成）

## 1. 项目入口与常用命令

项目为 Spring Boot 后端，默认地址为 `http://localhost:8888`，所有业务接口统一使用 `/api` 前缀。

```powershell
# 当前可用：运行测试
mvn test

# 当前可用：启动应用
mvn spring-boot:run
```

`mvnw.cmd` 是项目 Maven Wrapper。请在 `backend/aivista` 目录优先使用 `.\mvnw.cmd test` 和 `.\mvnw.cmd spring-boot:run`，以统一团队使用的 Maven 版本。

## 2. 已有基础能力

| 能力 | 使用方式 / 约定 |
| --- | --- |
| 健康检查 | `GET /api/health`；用于本地启动后确认服务可用。 |
| 接口文档 | Knife4j：`/api/doc.html`；Swagger UI：`/api/swagger-ui.html`；OpenAPI JSON：`/api/v3/api-docs`。新增 Controller 接口时应补充 OpenAPI 注解。 |
| 统一响应 | Controller 成功响应使用 `ResponseUtils.success(data)`，结构固定为 `code`、`message`、`data`。 |
| 业务异常 | 可预期的业务失败使用 `BusinessException` 和 `ErrorCode` 抛出，由全局异常处理器统一转换为 HTTP 响应。 |
| 参数校验 | 请求 DTO 使用 Jakarta Validation 注解；校验失败由全局异常处理器统一返回。 |
| 跨域 | 开发环境允许 `http://localhost:5173`；在 `application.yaml` 的 `app.cors.allowed-origins` 中维护来源。 |
| 日志 | 未处理异常由全局异常处理器记录；业务代码应使用 SLF4J 记录必要上下文，避免记录密码、令牌等敏感信息。 |

## 3. 代码位置

| 位置 | 职责 |
| --- | --- |
| `controller` | HTTP 接口层。 |
| `common/response` | 统一响应对象与创建工具。 |
| `common/exception` | 错误码、业务异常和全局异常处理。 |
| `common/request` | 通用请求 DTO 基类。 |
| `config` | 跨域和 OpenAPI 等工程配置。 |
| `application.yaml` | 服务端口、上下文路径、跨域和接口文档配置。 |

## 4. 开发约定

- 每次开发同步修改代码和相关文档。
- 新模块开始前，先补充该模块的详细设计文档；数据库表、API、权限和测试规则在该文档中确定。
- 新增接口遵循统一响应和异常处理约定，不在 Controller 中自行拼装错误响应。
- 迭代状态变化时，同步更新《后端项目开发文档》。
